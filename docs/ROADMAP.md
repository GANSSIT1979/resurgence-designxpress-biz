# ROADMAP

## Near-Term Priorities

### 1. Stabilization
- finish schema-aligned page conversions
- finish API consistency across admin, sponsor, and cashier modules
- complete premium CSS and shared component adoption
- reduce remaining route-by-route inconsistencies

### 2. Workflow Intelligence Layer
- smarter filters and search
- saved views
- row actions
- module-specific summaries
- better list bulk actions

### 3. Reporting Layer
- chart improvements
- export flows
- saved report snapshots
- print-ready finance and sponsor reports

### 4. File and Media Strategy
- move uploads from local filesystem to durable object storage for production
- add image validation and optimization
- add gallery event management improvements

### 5. AI and Support Expansion
- stabilize optional AI support mode
- richer chat lead capture workflows
- admin visibility into support escalations
- better support analytics

## Medium-Term Goals

- richer sponsor billing and payment visibility
- better cashier aging and reconciliation tools
- module-level audit logs
- notification center improvements
- email queue operational tooling
- advanced creator and campaign analytics

## Long-Term Goals

- fully production-grade storage strategy
- stronger automated test coverage
- background job processing
- workflow automation
- deeper CRM-style sponsor lifecycle management
- mobile-first dashboard optimization for field use

## Product Direction

The long-term direction is a single platform where:

- public users discover and inquire
- sponsors apply and track commitments
- admins manage content and business operations
- cashiers handle financial documents and reporting
- support can operate with or without AI
